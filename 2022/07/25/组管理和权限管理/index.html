组管理和权限管理

发表于2022-07-25|更新于2022-07-26|LINUX
|阅读量:

组基本介绍

在LINUX中的每个用户必须属于一个组，不能独立于组外。在linux中每个文件有所有者，所在组，其他组的概念。

所有者

概念：一般为文件的创建者

查看文件的所有者

ls

修改文件所有者

指令： chown 用户名 文件名

所在组

组的创建

基本指令：groupadd 组名

创建一个组monster，创建一个用户dog并放入到monster中

查看所在组

指令： ll

修改所在组 （文件de）

基本指令：chgrp 组名 文件名

其它组

除文件的所有者和所在组的用户外，系统其它用户都是文件的其他组

查看当前已经存在的组：

cat /etc/group

改变用户所在

usermod -g 新组名 用户名

usermod -d 目录名 用户名 改变该用户登录的初始目录 （用户需要右进入到新目录的权限）

权限的基本介绍

文件类型：

  • l : 连接 （快捷方式）
  • d : 目录 （文件夹）
  • c : 字符设备文件，鼠标，键盘
  • b : 设备： 硬盘

举例：

绿色的文件代表赋予了可执行

修改权限 chmod

  • 第一种方式 + 、- 、= 变更权限

    • u: 所有者 g: 所有组 o: 其他组 a: 全部（u，g, o 的综合）
    • chmod u = rwx, g = rwx, o = rwx 文件 / 目录名
    • chmod o + w 文件 / 目录名
    • chmod a - x 文件 / 目录名

  • 通过数字更改权限

    • r = 4 w = 2 x = 1 rwx = 4+2+1 = 7
    • chmod u=rwx,g=rx,o=x 文件 / 目录名 相当于 chmod 751 文件 / 目录名

修改文件所有者 chown

  • chown newowner 文件 / 目录 （改变所有者）
  • chown newowner:newgroup 文件 / 目录 （****改变所有者和所在组）
  • -R 如果是目录，则使其下所有子文件或目录递归生效
  • chgrp newgroup 文件 / 目录 (改变所在组)
